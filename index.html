Midum is an agentic framework built for local AI desktop assistance. It can help with code, UI automation, repetitive tasks, browsing, research, and file management — all driven by a tool-calling AI agent that runs on your machine.

The Problem: AI is Trapped in the Browser

Most AI assistants can talk about a task, but they can't actually do it on your machine.

⚠️

Chatbots Can't Act

Traditional AI assistants are isolated web forms. They give instructions and generate text, but leave the heavy lifting of clicking, typing, file management, and workflow execution to human users.

Result: High user fatigue & manual overhead

🔒

Privacy & Cloud Lock-in

Most AI automation relies on sensitive desktop data being sent to proprietary cloud endpoints, raising security alarms for enterprises, businesses, and privacy-conscious users.

Result: Enterprise adoption barriers

Where Midum Sits

Midum is the middle layer: it takes what you ask for, decides what the model should reason about, and turns that reasoning into actual actions on your computer.

●You describe a goal — nothing more than plain language.
●Midum is the loop in between: it plans, calls a tool, reads the result, and repeats.
●The model only reasons; the computer is where the action actually happens.

What Midum Can Do

Midum is built around a large tool-calling loop, so it isn't limited to chatting — it can actually take action on your computer.

🖥️ Desktop & UI Automation ›
  • Control the desktop through UI Automation (UIA) — click buttons, type into fields, navigate menus, and interact with native Windows applications.
  • Scan and inspect application UI trees (manual_scan_app_layouts, manual_inspect_app_subtree) before acting.
  • Click specific UI elements directly by control ID/name (click_ui_element).
  • Take an automatic snapshot of the active window and act on the elements it finds, planning a sequence of interactions.
  • List all active windows (list_active_windows).
  • Fallback vision-based control when UIA isn't reliable enough: screen snapshots, OCR-based text finding and clicking.
  • Type arbitrary text into focused fields (type_text).
  • Global Ctrl+Q abort hotkey to immediately interrupt a running task.
🌐 Browser Automation ›
  • Full browser page reading and interaction via the Chrome DevTools Protocol (remote debugging) for navigation, clicking, and typing on complex pages.
  • Read the current page (read_browser_page), list open tabs (list_browser_tabs), run JavaScript in the page context (run_js_in_browser), and open new URLs (open_url).
  • Falls back gracefully to UIA-based control when remote debugging isn't enabled.
  • Live internet search (search_internet) via DuckDuckGo, with indexed result opening.
📁 Files & Documents ›
  • Read and write local files directly (read_local_file, write_local_file, append_local_file).
  • Smart chunked reading for large files, supporting .txt, .md, .py, .json, .csv, .html, .pdf (via pymupdf), and .docx (via mammoth).
  • Generate Word documents from Markdown-style text (write_docx_file, via python-docx).
  • Browse and search the filesystem: list directories, open files/folders, fuzzy-find by name.
🧠 Memory, Goals & Planning ›
  • Persistent memory Midum can read and update across a conversation (update_memory).
  • Goal tracking — set, remember, and clear an active objective (set_current_goal).
  • A dedicated response scratchpad for multi-step tasks: Midum writes a plan first, appends progress notes, and reads it back to assemble a final answer.
📚 Skills & Knowledge Bases ›
  • Create, load, and list reusable skills — self-contained instructions for recurring tasks.
  • Create, read, and list custom domain knowledge files that act as a long-term reference library.
  • Manage a persistent instruction set that shapes Midum's behavior.
  • Manage saved filesystem paths/shortcuts for quick access.
  • Indexed listing variants keep large libraries easy to browse without flooding context.
🤖 Multi-Model & Multi-Provider Support ›

Midum isn't tied to a single model or provider — it can run its primary reasoning loop on any of the following, and can also consult or delegate sub-tasks to several side-by-side:

  • Ollama — fully local, no internet required. Now optional.
  • Gemini — via the official Gemini API, or via gemini_webapi Google account cookie sign-in (no API key).
  • OpenRouter — any OpenRouter-hosted model, with automatic fallback across a configurable list if one is rate-limited.
  • GroqCloud — fast inference on models like llama-3.3-70b-versatile and qwen/qwen3-32b, with a free tier and automatic fallback.
  • Ollama Cloud — the same Ollama client pointed at Ollama's hosted GPUs, no local GPU or download required, with fallback across a configurable model list.
  • Cross-consultation and delegation tools regardless of primary provider, plus tools to list and switch models per-provider on the fly.
🔗 Flows (visual automation builder) ›

The GUI's Flows tab is a node-graph editor (built on Drawflow) for building repeatable, shareable automations without writing Python by hand:

  • Drag out Start/End markers, native-tool nodes, MCP-server tool nodes, and nodes that call other saved flows, then wire them together.
  • Variable nodes hold a named piece of data for downstream nodes.
  • Logic nodes — if/else branching and for-each loops.
  • AI nodes — Prompt AI, Ask AI to Summarize, Ask AI to Choose — call out to the currently configured provider mid-run.
  • Saving a flow compiles the graph into a real Python function, safe to re-save repeatedly, with its graph persisted so it can be reloaded later.
  • Every saved flow is discoverable and callable as a tool, the same two-step shape as calling an MCP tool.
  • Promoted flows get their own always-on tool schema, callable directly by name.
  • Scheduling — saved flows can run automatically (once, interval, daily, weekly) while the app is open; schedule configuration persists across restarts.
🔌 MCP (Model Context Protocol) Support ›
  • Connect Midum to external MCP servers over stdio or HTTP/SSE.
  • List connected servers and their available tools.
  • Call any tool exposed by a connected MCP server directly.
  • Works uniformly across every model provider.
🎨 Generation Tools ›
  • Generate images on request.
  • Generate flowcharts/diagrams.
🗣️ Interaction & Control ›
  • Midum can proactively ask for input: free text, a file path, explicit approval before a sensitive action, or a choice from a list.
  • A say tool for clean, direct responses.
  • Tool discovery on demand, keeping the active toolset lean for smaller/local models.
  • A wait tool for timed pauses mid-task.
🖼️ Midum Control Centre (GUI) ›

Running gui.py gives you the full Midum Control Centre, a desktop app (built with customtkinter) on top of the same agent engine, including:

  • A live chat interface with real-time activity/tool-call logging.
  • A Model tab to pick provider and model, with live list refreshing.
  • A Parameters tab showing live agent state.
  • A System Core tab for managing the persistent instruction set.
  • Knowledge and Skills tabs for browsing and creating each.
  • A Tools tab for inspecting native tools and promoting/demoting Flows and MCP tools.
  • An MCP tab for adding, viewing, and managing connected servers.
  • A Flows tab for the node-graph editor described above.
  • Full editing access to every underlying file Midum uses — skills, knowledge bases, memory, instructions, and more.

This is the recommended way to run Midum, since it exposes everything the CLI does plus direct file editing.

Setup Instructions

Step 1 — Get the project

Download the ZIP from releases and extract it into an empty folder, or clone the repo. This is a proper Python package, so all commands should be run from inside it:

cd midum_pkg

Step 2 — Install dependencies

The easiest way is via the included requirements.txt:

pip install -r requirements.txt

Or install everything explicitly:

pip install ollama pillow ddgs keyboard pymupdf mammoth python-docx rich pytesseract pywin32 uiautomation customtkinter google-genai requests mcp
pip install -U gemini_webapi
pip install -U browser-cookie3   # optional but recommended

Step 3 — Ollama (optional)

Ollama is optional. Midum can run its primary reasoning loop directly on Gemini instead, so you can skip to Step 6 if you'd rather not run anything locally. For a fully local, offline-capable setup:

ollama pull qwen2.5-coder:7b

Then update the Modelfile to your chosen model and apply it from the folder it's in:

ollama create midum -f ./Modelfile

Step 4 — Choose a provider

Open config.py and set MODEL_PROVIDER:

MODEL_PROVIDER = "ollama"       # local Ollama model
MODEL_PROVIDER = "gemini_api"   # official Gemini API (API key from aistudio.google.com)
MODEL_PROVIDER = "gemini_web"   # Gemini via browser-cookie sign-in, no API key
MODEL_PROVIDER = "openrouter"   # any OpenRouter-hosted model
MODEL_PROVIDER = "groq"         # GroqCloud (free tier, fast inference)
MODEL_PROVIDER = "ollama_cloud" # Ollama's own hosted GPUs, no local GPU needed

Fill in the matching model/key settings just below MODEL_PROVIDER for whichever provider you chose.

Step 5 — Secrets file

Create a local JSON file holding your API keys / cookies, so nothing is hardcoded in config.py:

python -c "import config; created = config.ensure_secrets_file(); print('Created:' if created else 'Already exists:', config.SECRETS_FILE)"

This writes a blank template to %LOCALAPPDATA%\JarvisSecrets\jarvis_secrets.json (Windows) or ~/.config/JarvisSecrets/jarvis_secrets.json (Linux):

{
  "GEMINI_API_KEY": "",
  "OPENROUTER_API_KEY": "",
  "GROQ_API_KEY": "",
  "OLLAMA_API_KEY": "",
  "GEMINI_SECURE_1PSID": "",
  "GEMINI_SECURE_1PSIDTS": ""
}

Fill in only the key(s) your chosen provider needs — a blank value is treated as "not configured", so the rest can stay empty. This file is also created automatically the first time you run main.py or gui.py.

Optional — OCR

Install Tesseract to enable OCR-based screen reading and clicking. Midum is fully functional without it — OCR is only used as a fallback for UI interaction.

Step 6 — Launch

Run gui.py for the full Midum Control Centre (recommended), or main.py for the CLI. Both run from inside the project folder:

python gui.py
# or
python main.py

Alternative Setup — Prebuilt Executable

Windows users can skip the full setup above and instead:

  1. Download midum.exe from the releases page and place it in its own empty folder (it creates storage/ and other supporting files alongside itself on first run).
  2. Launch it once and close it again — this creates the blank secrets template at %LOCALAPPDATA%\JarvisSecrets\jarvis_secrets.json. Open the file, fill in only the key(s) your chosen provider needs, and save it.
  3. Relaunch midum.exe and you're straight into the Midum Control Centre — no pip install, no Ollama, no IDE required.

Browser Support

For reliable browser page navigation and interaction, launch your browser with Remote Debugging enabled. Without it, Midum falls back to UI Automation (UIA), which works for many tasks but may be less reliable on complex web pages.

& "C:\Program Files\BraveSoftware\Brave-Browser\Application\brave.exe" --remote-debugging-port=9222 --remote-allow-origins=*
